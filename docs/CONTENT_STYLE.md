# Learning Content Style

ExamServer の基礎知識ページは、試験用語を並べる場所ではなく、初心者が次に何を見ればいいかを決める場所です。

## 読者

- PC操作はできるが、ネットワークやセキュリティは初学者。
- コマンド、HTTP、DNS、メモリなどの単語は聞いたことがあるかもしれないが、まだつながっていない。
- いきなりCTFへ行くと、何を見ているのか分からなくなる人を想定する。

## 文体

- 教科書調に寄せすぎない。
- 「先輩が横で説明する」くらいの距離で書く。
- 完璧な並列構造にしない。段落の長さは揃えすぎない。
- 強い言葉を連発しない。「悪くない」「最初はこれで十分」くらいの中間温度を混ぜる。
- 見出しはテーマ名にする。「OSI参照モデルが重要な理由」のような命題型見出しを増やしすぎない。

## AI臭を避ける

- 「現代社会において」「重要です」「本質的です」で逃げない。
- 「AではなくBだ」の反転構文を連発しない。
- 「それぞれ順に見ていきましょう」は削る。
- 「解像度」「文脈」「本質」「思想」「設計」を雑に撒かない。
- 事実を調べて書くときは、出典を置く。直接確認していないものは断定しすぎない。

## 引用と画像

- 図、絵、写真は公的/公式/ライセンス確認済みを優先する。
- 埋め込む画像には、出典URLとライセンスメモを必ず付ける。
- ライセンスが分からない画像は埋め込まない。必要ならリンクだけにする。
- コードでそれっぽい図を自作するより、信頼できる図を引用して、本文で初心者向けに噛み砕く。

## ページ構成

- 冒頭で「このページで何が分かるか」を短く置く。
- 本文では、用語の定義だけでなく「どこで使う考え方か」を書く。
- 最後は確認演習か知識マップに戻す。
- バッジ、ランキング、過剰な進捗率は入れない。現在地と次の一歩だけでよい。

## 数を数えるとはどういうことか

人間は普段、0から9までの10個の記号で数を書きます。

10本の指で数えてきた名残です。

9の次は記号が尽きるので、桁を一つ増やして「10」と書きます。

この「10個の記号を使い、尽きたら桁を増やす」という方式を**10進数**と呼びます。

コンピュータの内部はこの方式では動いていません。

CPU（コンピュータの計算を担う部品）は電気回路でできていて、扱えるのは電圧の高低、つまり「電流が流れている」か「流れていない」かの2状態だけです。

10段階の電圧を正確に区別する回路は作れますが、配線の長さやノイズで電圧がわずかにずれただけで誤判定が起きます。

2状態だけなら、多少の電圧のゆれがあっても「高いか低いか」は簡単に区別できます。

この物理的な制約から、コンピュータは数を2つの記号だけで表す方式を採用しています。

これが**2進数**です。

## 2進数の書き方

2進数は0と1の2個の記号だけを使います。

10進数と同じく、記号が尽きたら桁を増やします。

```
2進数   10進数
0       0
1       1
10      2
11      3
100     4
101     5
110     6
111     7
1000    8
```

10進数では「9の次で桁が増える」のに対し、2進数では「1の次で桁が増える」だけの違いです。

仕組みは同じです。

10進数の各桁は1, 10, 100, 1000...（10の累乗）の重みを持ちます。

2進数の各桁は1, 2, 4, 8...（2の累乗）の重みを持ちます。

たとえば2進数の `1101` を10進数に変換するには、1が立っている桁の重みを足します。

```
1101
= 1×2³ + 1×2² + 0×2¹ + 1×2⁰
= 8 + 4 + 0 + 1
= 13
```

逆に10進数の13を2進数に変換するには、2で割った商と余りを繰り返し求めます。

```
13 ÷ 2 = 6 余り 1
 6 ÷ 2 = 3 余り 0
 3 ÷ 2 = 1 余り 1
 1 ÷ 2 = 0 余り 1
```

余りを下から上に読むと `1101` になります。

これが先ほどの変換と一致することは、上の式で確認済みです。

## ビットとバイト

2進数の1桁、つまり0か1のどちらか一方を**ビット**（bit）と呼びます。

「binary digit」（2進数の桁）を短縮した語です。

ビットはコンピュータが扱う情報の最小単位です。

ビットを8個まとめた単位を**バイト**（byte）と呼びます。

```
1バイト = 8ビット
```

8という数に深い理由はありません。

歴史上いくつかの候補（6ビット、9ビットなど）が使われた時期もありましたが、英語のアルファベットや記号、制御コードを1単位で収められる大きさとして8ビットが定着しました。

8ビットで表現できる組み合わせの数は、各ビットが0か1の2通りなので、8個分掛け合わせて求まります。

```
2⁸ = 256通り
```

0から数えるので、1バイトで表せる値は0から255までです。

<Callout title="最大値が255になる理由">
  256通りの値があるとき、0を含めて数えるので最大値は256ではなく255になります（0, 1, 2, ..., 255で256個）。
</Callout>

ビットとバイトは似た文脈で出てきますが、用途が違います。

ファイルサイズは「バイト」単位（B, KB, MBなど）で表すのが通例です。

一方、ネットワークの通信速度は「ビット」単位（bps, Mbpsなど）で表すのが通例です。

8倍の差があるので、「1Gbpsの回線」と「1GBのファイル」を同じ尺度で比べるとつまずきます。

回線速度をバイト換算するときは8で割る必要があります。

## 16進数が使われる理由

2進数はコンピュータの動作に直結していますが、人間が読み書きするには桁数が多すぎます。

たとえば32個のビットがすべて1だったとします。

```
11111111111111111111111111111111
```

1の個数を数え間違えないとも限りません。

ここで使われるのが**16進数**です。

0から9の数字10個に加えて、A, B, C, D, E, Fの6文字を使い、合計16個の記号で数を表します。

```
2進数    16進数
0000  →  0
0001  →  1
0010  →  2
0011  →  3
0100  →  4
0101  →  5
0110  →  6
0111  →  7
1000  →  8
1001  →  9
1010  →  A
1011  →  B
1100  →  C
1101  →  D
1110  →  E
1111  →  F
```

16という数を選んだ理由は、2の4乗がちょうど16になることにあります。

```
2⁴ = 16
```

2進数の4桁（4ビット）が、16進数のちょうど1桁に対応します。

このため、2進数を16進数に変換するときは、4ビットずつ区切って表の通りに置き換えるだけで済みます。

先ほどの32ビットの並びを4ビットごとに区切ってみます。

```
1111 1111 1111 1111 1111 1111 1111 1111   （2進数、4ビットごとに区切った）
 F    F    F    F    F    F    F    F     （16進数）
```

32文字だった2進数表記が、16進数では8文字になりました。

桁数が4分の1になるので、人間が読み書きするときの負担が大きく減ります。

CPUの内部表現が2進数のままであることに変わりはありません。

16進数は、その2進数のビット列を人間が扱いやすい形に書き直したものです。

## 0xという接頭辞の意味

セキュリティや低レイヤーの分野の資料を読むと、次のような数値表記によく出会います。

```
0x48656c6c6f
0x1f
0xdeadbeef
```

先頭の `0x` は、「この後に続く数字は16進数として読んでください」という合図です。

10進数の「10」と16進数の「10」は値が違うので（10進数の10は16進数では`a`）、何の合図もないと読み手が混乱します。

`0x` という接頭辞を付けることで、その混乱を避けています。

```
0x10 = 16進数の「10」= 10進数の「16」
0x20 = 16進数の「20」= 10進数の「32」
0xff = 16進数の「ff」= 10進数の「255」
```

プログラミング言語の多くは、16進数以外の記数法にも接頭辞を用意しています。

8進数には `0o`、2進数には `0b` を使う言語が一般的です。

```python
print(0x10)   # 16進数の10 → 出力: 16
print(0o10)   # 8進数の10  → 出力: 8
print(0b10)   # 2進数の10  → 出力: 2
```

## \xNNという記法

16進数の表記には、もう一つよく見る形があります。

```
\x41\x42\x43
```

これは1バイトずつの値を16進数で並べた**バイト列**の表記です。

`\x` の直後の2桁が、1バイト分の値を16進数で表しています。

```
\x41 = 16進数の「41」= 10進数の「65」
\x42 = 16進数の「42」= 10進数の「66」
\x43 = 16進数の「43」= 10進数の「67」
```

この数値を**ASCII文字コード表**（文字と数値の対応表）で照合すると、65は「A」、66は「B」、67は「C」に対応します。

つまり `\x41\x42\x43` は、文字列「ABC」をバイト単位の数値で書き表したものです。

<Callout title="ASCII文字コード表">
  すべてを覚える必要はありません。`0x41`が'A'、`0x20`がスペース、`0x0a`が改行であることだけ知っていれば、ファイル解析の場面で読みやすさが大きく変わります。
</Callout>

文字列「Hello」を同じ記法で書くと、次のようになります。

```
H → 0x48 → \x48
e → 0x65 → \x65
l → 0x6c → \x6c
l → 0x6c → \x6c
o → 0x6f → \x6f

\x48\x65\x6c\x6c\x6f
```

バイナリエディタ（ファイルの内容をバイト単位で表示し編集するソフト）でファイルを開いたとき、画面に現れる数値の並びは、この `\x` 記法と同じものです。

プログラムが処理しているのは「H」という文字そのものではなく、`0x48`という1バイトの数値です。

文字として表示するか、数値として表示するかは、表示する側（テキストエディタかバイナリエディタか）の選択に過ぎません。

この対応関係を踏まえておくと、ファイルフォーマットの解析やシェルコード（攻撃用に作られた機械語の断片）の読み取りで出てくる `\x` 表記が、見慣れない記号の集まりではなく、文字や数値の言い換えとして読めるようになります。

## この章のまとめ

**2進数**は0と1だけで数を表す記数法です。

コンピュータの電気回路が2状態しか区別できないことに対応しています。

**ビット**は0か1の1個分の情報、**バイト**はビット8個分の単位です。

ファイルサイズはバイト単位、通信速度はビット単位で表すのが通例なので、両者を混同しないようにします。

**16進数**は0から9とA からFまでの16個の記号を使う記数法です。

2進数の4ビットがちょうど16進数の1桁に対応するため、2進数を短く書き直す手段として使われます。

**0x**という接頭辞は、続く数字が16進数であることを示す合図です。

**\xNN**という記法は、1バイトの値を16進数2桁で表したバイト列の表記です。

ASCII文字コード表と組み合わせれば、`\x41`が文字「A」を指すといった対応がそのまま読めます。

## 「chmod 777にすれば直る」という解決策がなぜ危険なのか

プログラムを実行したときに、こういうエラーが出ることがあります。

```
Permission denied
```

検索すると、よくこの解決策が出てきます。

```
chmod 777 filename
```

実際、これを実行するとエラーは消え、プログラムは動き出します。

ただ、`chmod 777` が何を意味しているかを理解せずに使うと、誰でもそのファイルを書き換えられる状態を作ってしまいます。

この記事では、Linuxの**パーミッション**（権限）がどういう仕組みなのか、そして `chmod 777` がなぜ危険なのかを、順番に説明します。

## rwxという3つの操作

パーミッションは、3種類の操作を許可するかどうかの組み合わせで表されます。

**r**（read、読み取り）は、ファイルでは中身を見る権限、ディレクトリではその中のファイル一覧を表示する（`ls` する）権限です。

**w**（write、書き込み）は、ファイルでは編集や削除をする権限、ディレクトリではその中にファイルを作成したり削除したりする権限です。

**x**（execute、実行）は、ファイルでは実行ファイルとして実行する権限、ディレクトリではその中に「入る」（`cd` で移動する、その中のファイルにアクセスする）権限です。

ディレクトリの `x` が何を意味するかは、初めて学ぶときに混乱しやすい点です。

`x` がないディレクトリは、`ls` で一覧を見られても、その中のファイルにはアクセスできません。

実際にどう振る舞うかを見てみます。

```
r-x （読み取り＋実行）
ls /home/alice    一覧表示も中身へのアクセスもできる

r-- （読み取りのみ）
ls /home/alice    成功（一覧は見える）
cd /home/alice    Permission denied（中には入れない）

--x （実行のみ）
cd /home/alice    成功（中には入れる）
ls /home/alice    Permission denied（一覧は見えない）
```

「一覧は見えるのに中に入れない」状態と、「中に入れるのに一覧は見えない」状態が両方存在します。

直感に反するように見えますが、これがLinuxの仕様です。

## 権限は誰に対して設定されるのか

パーミッションは、3つの立場の人に対して、それぞれ独立に設定されます。

**u**（user）はファイルの所有者です。

**g**（group）はファイルが所属するグループに属するユーザーです。

**o**（other）はそれ以外のすべてのユーザーです。

たとえば、aliceが作成したファイル `secret.txt` のパーミッションが次のようになっているとします。

```
-rw-r--r-- 1 alice alice
```

これを読み解くと、所有者（u）は `rw-` で読み取りと書き込みができ、実行はできません。

グループ（g）は `r--` で読み取りだけができます。

その他（o）も `r--` で読み取りだけができます。

## 数値表記755、644、777の読み方

パーミッションを短く表記するために、`r` を4、`w` を2、`x` を1として、許可されている権限の数値を足し合わせる方法があります。

`rwx` であれば 4+2+1 で7、`rw-` であれば 4+2 で6、`r--` であれば4です。

`chmod` の数値表記は、所有者、グループ、その他の順に、この数値を3つ並べたものです。

```bash
chmod 755 script.sh
```

この `755` を分解すると、所有者は7（`rwx`）、グループは5（`r-x`）、その他は5（`r-x`）になります。

## chmodとchown、それぞれが変更するもの

パーミッションを変更するコマンドが `chmod`、所有者やグループを変更するコマンドが `chown` です。

```bash
chmod 644 config.txt
chmod 755 script.sh
chmod +x script.sh
chmod -w config.txt
```

`chmod 644` や `chmod 755` は数値でパーミッション全体を指定します。

`chmod +x` は、現在のパーミッションを保ったまま実行ビットだけを追加する指定です。

`chmod -w` は書き込み権限だけを削除する指定です。

```bash
chown alice config.txt
chown alice:developers config.txt
chown :developers config.txt
```

`chown alice` は所有者をaliceに変更します。

`chown alice:developers` は所有者をalice、グループをdevelopersに変更します。

`chown :developers` はグループだけをdevelopersに変更します。

<Callout title="chmodとchownにroot権限は必要か">
自分が所有しているファイルであれば、`chmod` でパーミッションを変更するのにroot権限は不要です。

`chown` で所有者を別のユーザーに変更する場合は、ファイルの所有権をその新しいユーザーに渡す操作になるため、root権限が必要です。
</Callout>

## ディレクトリの実行権限はなぜ必要なのか

ディレクトリに対する `x` 権限は「そのディレクトリの中に入る」（`cd` で移動する、その中のファイルにアクセスする）権限です。

これはファイルの実行権限とは別の意味を持つので、もう一度整理します。

```
drwx------ 2 alice alice    aliceだけがすべての操作ができる
drwxr-xr-x 2 alice alice    aliceは全権、他のユーザーは一覧表示と中身へのアクセスができる
dr-xr-xr-x 2 alice alice    aliceも他のユーザーも中身へのアクセスと実行はできるが、一覧表示はできない
```

3つ目の例は「ファイルの内容には直接アクセスできるが、ディレクトリに何が入っているかは一覧で確認できない」という状態です。

ファイル名を知っている人だけがアクセスできるようにしたい場合に使われます。

## chmod 777が危険な理由

ここまでの知識を踏まえて、`chmod 777` が何をしているかを確認します。

```bash
chmod 777 filename
```

この一行は、所有者、グループ、その他のすべてのユーザーに対して、読み取りと書き込みと実行のすべてを許可します。

```
-rwxrwxrwx 1 alice alice
```

このファイルにアクセスできるすべてのユーザーが、内容を書き換えられます。

そのファイルが実行ファイルであれば、悪意のあるコードに書き換えられた状態のまま実行される可能性があります。

そのファイルが秘密鍵であれば、誰でもその鍵の持ち主になりすますことができます。

「アップロードされたファイルにエラーが出たのでchmod 777を付けた」という対応は、典型的な失敗のパターンです。

複数のユーザーからアクセスできるようにしたいのであれば、その対象をグループにまとめて、グループ単位で権限を与える方法が適切です。

<Callout title="chmod 777の代わりに使える設定">
```
chmod 750 filename
chown alice:developers filename
```
この設定は、所有者（alice）が全権を持ち、developersグループに読み取りと実行を許可し、その他のユーザーには何も許可しません。

```
chmod 664 filename
```
この設定は、所有者とグループに読み取りと書き込みを許可し、その他には読み取りだけを許可します。

どちらを選ぶかは、共有したい操作の範囲によって決めます。
</Callout>

## setuidとsetgidという特殊な権限

パーミッションには、ここまで説明したr/w/x以外に、特殊なビットがあります。

ここでは概要だけを説明します。

**setuid**（数値表記では4000）は、実行ファイルに付けると、そのファイルを実行したユーザーに関わらず、ファイルの所有者の権限で実行されるという設定です。

`sudo` コマンドはこの仕組みの実例です。

`/usr/bin/sudo` の所有者はrootで、setuidビットが設定されています。

一般ユーザーが `sudo` を実行すると、実行している本人の権限ではなく、root権限でプログラムが動きます。

これが `sudo` によって権限を昇格できる理由です。

```
-rwsr-xr-x 1 root root
```

所有者のパーミッションの位置に `x` の代わりに `s` が表示されているのが、setuidビットの印です。

**setgid**も同様の仕組みで、ファイルの所属グループの権限で実行されます。

setuidとsetgidの詳しい挙動は、セキュリティに関する別の記事で扱います。

ここでは「実行ユーザーとは異なる権限でプログラムが動く仕組みがある」という事実を知っておけば十分です。

## ここまでのまとめ

パーミッションは、読み取り（r）、書き込み（w）、実行（x、ディレクトリでは「中に入る」権限）という3種類の操作を、所有者（u）、グループ（g）、その他（o）という3つの立場それぞれに対して設定する仕組みです。

数値表記では `r=4, w=2, x=1` を足し合わせ、`755` のように3つの数字で表します。

`chmod 777` は全ユーザーに読み取りと書き込みと実行のすべてを許可する設定で、ファイルの改ざんや機密情報の漏洩につながるため避けるべきです。

複数ユーザーでの共有が必要な場合は、グループを使って必要最小限の権限だけを与える方法が適切です。

## 同じサーバーで3つのサービスが同時に動いている

SSHでサーバーにログインしたとき、同じそのサーバーでWebサービスも動いている——というのはよくある話です。

「1台のサーバー = 1つの宛先（IPアドレス）」なら、外から届いたパケットはどうやって「これはWebへ」「これはSSHへ」と振り分けられるのか？

ここで出てくるのが **ポート** です。

## ポートは「扉の番号」

IPアドレスが「どのビル（サーバー）か」を示すなら、ポートは「そのビルの何番の扉か」を示す番号です。0〜65535の整数で、各サービスが1つ（または複数）のポートで「待ち受け」ます。

接続が来たとき、OSはポート番号を見て「これを80番で待っているWebサーバーへ渡す」「これを22番で待っているSSHへ渡す」と判断します。

`93.184.216.34:80` と書いたら、IPアドレス `93.184.216.34` の **80番ポート** への接続という意味です。URLのデフォルトポート（HTTPなら80、HTTPSなら443）は省略できるので、普段は意識しませんが、実際には毎回ポート番号を指定して通信しています。

## ウェルノウンポートの「常識」と、その実態

0〜1023番は **ウェルノウンポート**（Well-known ports）と呼ばれ、代表的なサービスに割り当てられています。

| ポート | サービス | 用途 |
|---|---|---|
| 22 | SSH | 安全なリモートログイン |
| 25 | SMTP | メール送信 |
| 53 | DNS | 名前解決（前の回で出てきましたね） |
| 80 | HTTP | Web |
| 443 | HTTPS | TLSを使うWeb |

ここで一個、よくある誤解に突っ込んでおきます。

「80番＝HTTP」というのは**慣習であって、技術的な制約ではない**んです。HTTPサーバーを8080番で動かすことも、3000番で動かすことも普通にあります。開発環境で `localhost:3000` にアクセスするのを見たことがあると思いますが、あれは3000番で待ち受けているサーバーへの接続です。Next.jsはデフォルト3000番、Djangoは8000番、Railsは3000番——フレームワークによってデフォルトはバラバラ。

ブラウザが80番（HTTPSなら443番）を省略できるのは「デフォルトとして使うポートをあらかじめ取り決めてある」から。`http://example.com:80/` と書いても `http://example.com/` と書いても同じです。

## 「待ち受ける」とはどういうことか

アプリケーションがポートで **リッスン**（listen）するとは何か、もう少し解像度を上げます。

SSHデーモンが起動すると、OSに「22番ポートで接続を待ちます」と登録します。これがリッスン。外から22番へ接続が来たら、OSがそれをSSHデーモンへ渡す。

今どのポートが待ち受け中かはコマンドで確認できます。

<Callout title="使用中のポートを確認する">
  Windows なら `netstat -an` をコマンドプロンプトで打つと、アクティブな接続と待ち受けポートの一覧が出ます。`LISTENING` の行が「待ち受け中」です。Linux なら `ss -tuln` が見やすい（Linuxのシェル回で改めて出てきます）。
</Callout>

## ファイアウォールはポートの門番

理解しておかないと、トラブル発生時に詰まる。

**ファイアウォール**はパケットをフィルタリングする仕組みで、多くの場合ポート単位で「通すか通さないか」を制御します。

「外から22番ポートは遮断する」——これでSSHの外部接続を防げます。「80番と443番だけ通す」——Webだけ公開してその他は閉じる、という設定です。

  ファイアウォールはポート単位の門番。開けていいポートだけ許可し、残りは閉じるのが基本。

クラウドでも同じ考え方が使われています。AWSのセキュリティグループ、GCPのファイアウォールルール、Azureのネットワークセキュリティグループ——どれもポート単位でインバウンド・アウトバウンドを制御します。「どのポートを外に開けるか」はサーバーを公開するときの基本的なチェック項目です。

## ここで押さえること

- ポートは0〜65535の番号。IPが「どのサーバーか」を示し、ポートが「そのサーバーのどのサービスへか」を示す
- 0〜1023は**ウェルノウンポート**——SSH:22、DNS:53、HTTP:80、HTTPS:443 など
- 「80番＝HTTP」は慣習であって技術的な制約ではない。開発環境では8080番や3000番なども普通
- **ファイアウォール**はポート単位でトラフィックを制御する。公開するポートを最小限にするのがセキュリティの基本
